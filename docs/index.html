Project Overview

What is NavRL Bench?

NavRL Bench is a robotics navigation project that combines reinforcement learning training and Nav2 controller benchmarking.

The project trains RL-based local navigation policies from scratch using IsaacLab / Isaac Sim. The trained policies are then integrated into a TurtleBot3 navigation pipeline and compared against classical Nav2 local controllers including DWB, MPPI, and Regulated Pure Pursuit.

Problem Statement

Why train an RL local controller?

Classical Nav2 controllers are reliable and widely used for mobile robot navigation. However, in dynamic obstacle scenarios, they can become conservative, stop frequently, or struggle to produce adaptive local avoidance behavior.

This project investigates whether reinforcement learning policies trained from scratch can learn more adaptive local navigation behavior and how their performance compares with established Nav2 controllers under the same test conditions.

RL Training Module

Training policies from scratch

Training Platform

RL policies are trained in simulation using IsaacLab / Isaac Sim with a TurtleBot3-based navigation environment.

Learning Algorithm

The initial policy training focuses on PPO. Additional RL algorithms can be added later for extended comparison.

Policy Output

The trained policy outputs velocity commands that can be used by the robot as local navigation actions.

Possible Observations

  • Robot linear and angular velocity
  • Goal direction and goal distance
  • Laser scan or local obstacle representation
  • Previous action
  • Local path information if used

Possible Rewards

  • Reward for moving toward the goal
  • Reward for reaching the goal
  • Penalty for collision
  • Penalty for unsafe obstacle distance
  • Penalty for excessive stopping
  • Penalty for jerky velocity commands

The exact observation space, action space, reward design, and training configuration will be documented as part of the implementation.

Controllers Compared

Classical baselines vs trained RL policy

DWB Controller

A classical Nav2 local controller based on the Dynamic Window Approach. It generates velocity commands by evaluating possible robot motions using critic functions.

MPPI Controller

A predictive sampling-based controller that evaluates multiple candidate trajectories and selects the best control action based on cost.

Regulated Pure Pursuit

A path tracking controller that follows the global path while regulating speed based on curvature and collision constraints.

Trained RL Policy

A reinforcement learning based local controller trained from scratch in IsaacLab / Isaac Sim and integrated into the navigation pipeline.

Benchmark Principle

No unfair comparison

Classical controllers are used as available Nav2 baselines. RL policies are trained from scratch by this project and then evaluated under the same test setup.

The project does not assume that RL will always perform better. The goal is to measure where classical control performs well, where it struggles, and whether trained RL policies provide better adaptability in dynamic obstacle scenarios.
